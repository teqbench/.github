# Support

Thanks for using a TeqBench project. This document explains where to direct different kinds of requests.

## Reporting a Bug

Open a bug report in the **affected repository** using the **Bug Report** issue template.

Before filing, please:

- Search existing open and closed issues for a duplicate.
- Confirm you can reproduce the issue on the latest published version.
- Include the information requested in the template (environment, reproduction steps, expected vs actual behaviour).

Bug reports are the primary supported channel. Reports filed on the wrong repository will be closed with a pointer to the correct one.

## Requesting a Feature

Open a feature request in the affected repository using the **Feature Request** issue template. Describe the problem you are trying to solve, not just the solution you have in mind — that context helps prioritisation.

Feature requests are evaluated on a best-effort basis and may be declined, deferred, or accepted with modifications.

## Reporting a Security Vulnerability

**Do not open a public issue.** See [SECURITY.md ↗](SECURITY.md) for the private reporting process.

## Asking a Usage Question

Usage questions are **not** supported via GitHub issues. Issues are reserved for reproducible defects and concrete feature proposals.

For questions about how to use a package, consult:

- The repository's `README.md`
- Any [Storybook ↗](https://storybook.js.org) or generated API documentation published alongside the package
- The source code and tests for worked examples

## Contributing

To contribute a fix, improvement, or new feature, see [CONTRIBUTING.md ↗](CONTRIBUTING.md).
